Nondeterministic Finite Automata

Nondeterminism gives a machine multiple options for its moves.

In a nondeterministic finite automaton (NFA), for each state there can be zero, one, two, or more transitions corresponding to a particular symbol.

If an NFA reaches a state with more than one possible transition for the input symbol, it branches.

If an NFA reaches a state where there is no valid transition, that branch dies.

NFA Acceptance

An NFA accepts the input string if there exists some choice of transitions that ends in an accept state.

One accepting branch suffices for acceptance; the NFA rejects only if all branches reject.

This is a model of computation. We write DFA for deterministic finite automaton; if the type doesn’t matter, we write FA.

Nondeterminism as "Guess and Verify"

There are many views of nondeterminism; one is “guess and verify”.

Idea: The NFA “guesses” the right next state (clairvoyantly), but it must “check” its guesses by following legal transitions.

Nondeterminism via Computation Tree

Think of nondeterminism as a tree growing downward; children are possible successor states.

The input is accepted exactly when one branch ends in an accept state.

More Nondeterminism: \$\varepsilon\$-transitions

We also allow \$\varepsilon\$-transitions: arrows labeled with the empty string.

These allow the NFA to change state without consuming an input symbol.

Formal Definition

A Nondeterministic Finite Automaton is a 5-tuple \$(Q,\ \Sigma,\ q_0,\ T,\ \delta)\$, where (as before):

  • \$Q\$ is a finite set of states;

  • \$\Sigma\$ is an alphabet of input symbols (binary allows inputs \$0\$ and \$1\$);

  • \$q_0\$ is the start (initial) state;

  • \$T \subseteq Q\$ is the set of accept states;

  • \$\delta\$ is a transition function.

Now the transition function specifies a set of states rather than a single state:

it maps \$Q \times \Sigma\$ to {subsets of \$Q\$}.

Conversions Among FAs and REs

We describe algorithms that convert among NFAs, DFAs, and REs.

Conversion From RE to NFA (recursive idea): 1. If RE is the empty string, output the trivial NFA. 2. If RE is a single symbol, output the trivial NFA. 3. If RE has form \$A + B\$, combine NFAs for \$A\$ and \$B\$. 4. If RE has form \$AB\$, concatenate NFAs for \$A\$ and \$B\$. 5. If RE has form \$A^{\ast}\$, extend the NFA for \$A\$.

From NFA to DFA: Subset Construction

Converting NFA to DFA uses the subset construction. To simulate an NFA on a string, keep track at each step of the set of states the NFA could be in. The next set depends only on the current set and the next input symbol.

Example: Simulating an NFA

Consider \$10100\$ as input to this NFA:

\$\{A\} \rightarrow 1\{A,C\} \rightarrow 0\{A,B\} \rightarrow 1\{A,C\} \rightarrow 0\{A,B\} \rightarrow 0\{A,B,D\}\$

Accepts \$10100\$ because it can be in an accept state after the last symbol.

Conversion from NFA to DFA

From NFA (without \$\varepsilon\$-transitions) to DFA:

  1. Each DFA state is a set of original states.

  2. Start state is \$q_0\$.

  3. While some DFA state is missing a transition: compute the transition by combining possibilities for each member state and symbol.

  4. Any DFA state (set) that contains an original accept state is accepting.

Need Closure for \$\varepsilon\$-Transitions

With \$\varepsilon\$-transitions:

  1. The DFA start state is the \$\varepsilon\$-closure of the original start.

  2. After each move on a symbol, take the \$\varepsilon\$-closure of the destination set.

Conversion From FA to RE

We now convert from FA to RE.

One approach: a generalized FA (GFA), where each transition is an RE. Build a series of GFAs; at each step, remove one non-start/non-accept state and replace it with equivalent transitions.

Removing a State

Say there is: * transition a from state 1 to 2, * transition b from 2 to 2, and * transition c from 2 to 3.

Same effect by a single transition \$ab^{\ast}c\$ from 1 to 3.

You must consider all transitions in and out of the removed state simultaneously.

Conversion From FA to RE: Summary

Assume a unique accept state, no transitions out of accept, and none into start.

Conversion steps:
1. Convert to the right form.
2. While more than two states remain, remove one and add equivalent transitions.
3. Read the final RE off the remaining transition.

Summary

Kleene’s theorem: the following are equivalent for a language:

  • there is an FA for it;

  • there is an NFA for it; and

  • there is an RE for it.

The proof yields algorithms to convert between forms; NFA→DFA uses subset construction.
