Regular Grammars

We have observed that every regular language has a CFG. In fact, a regular language (without \$\varepsilon\$) can be generated by a special form:

A regular grammar is one where each production has the form

\$A \rightarrow bC\$ or \$A \rightarrow a\$

(where \$a\$ and \$b\$ are terminals and \$C\$ is a variable).

Theorem. Every regular language is generated by a regular grammar.

Construction of Regular Grammar

Start with the DFA of the language. Introduce one variable per state.

  • For each transition, add a production: if \$\delta(A,x)=B\$, then add \$A \rightarrow xB\$.

  • For each transition that ends in an accepting state, also add:

    • if \$B\$ is an accepting state in the above case, also add \$A \rightarrow x\$.

  • The start variable is the initial state of the DFA.

Example

Consider the RE \$(11+00)^{\ast}11\$. Here is the NFA.

This produces the following regular grammar:

  • \$S \rightarrow 0C | 1A\$

  • \$A \rightarrow 1B | 1\$

  • \$B \rightarrow 0C | 1A\$

  • \$C \rightarrow 0S\$

Unrestricted & Context-Sensitive Grammars

In unrestricted grammars, the productions have the form \$u \rightarrow v\$ where \$u\$ and \$v\$ are arbitrary strings of terminals/variables.

In context-sensitive grammars, the productions have the form \$xAz \rightarrow xyz\$ where \$x\$, \$y\$, \$z\$ are strings of terminals/variables and \$A\$ is a variable.

Context-Sensitive Example

A CSG for \$0^n 1^n 2^n\$ is not obvious!

\$S \rightarrow 0BS2 | 012\$

\$B0 \rightarrow 0B\$

\$B1 \rightarrow 11\$

Try to derive \$000111222\$. It is correct, but nothing trivial to obtain.

The Chomsky Hierarchy

Chomsky introduced the hierarchy in the study of natural languages:

  1. Unrestricted grammars

  2. Context-sensitive grammars

  3. Context-free grammars

  4. Regular grammars

We have already seen: regular grammars ⇔ FAs and CFGs ⇔ PDAs. We will later see machines for the other two types.

Simplifying CFGs: Usable & Nullable Variables

A variable is usable if it generates some string of terminals.

A variable is nullable if it generates the empty string.

Example. Below, A and B are usable, but only B is nullable:

\$A \rightarrow 0A | 1B | 2C\$

\$B \rightarrow 0B | \varepsilon\$

\$C \rightarrow 1C\$

Summary for Regular Grammars

A regular grammar has all productions in the format

\$A \rightarrow bC\$

or

\$A \rightarrow a\$

The Chomsky hierarchy also includes context-sensitive and unrestricted grammars.

CFGs and PDAs are Equivalent

We will prove that a language is generated by some CFG if and only if it is accepted by some PDA.

The proof consists of two conversions.

Conversion from CFG to PDA

Idea: the PDA “guesses” the leftmost derivation and checks that it is correct (proof by simulation).

Each step of the derivation is a string. The machine guesses the next step, using only the stack.

The terminals to the left of the leftmost variable are not stored (they are matched with the input). Only the part of the string from the leftmost variable is kept on the stack, with the leftmost symbol at the top.

Operating the PDA

The PDA starts with the start variable on the stack. At each step:

  • If the top of the stack is a variable, the PDA chooses a production and replaces the variable with the RHS of that production.

  • If the top of the stack is a terminal, then it pops and reads a symbol from the input, checking for a match.

It accepts if and only if it reaches an empty stack at the end of the string. (Note: the PDA is not guaranteed to terminate.)

\$S \rightarrow 0S1S | 1S0S | \varepsilon\$

Formal Definition

From that perspective, a PDA is a 7-tuple

\$(Q,\Sigma,\Gamma,q_0,h_a,\Delta,\delta)\$ where:

  • \$Q\$ — finite set of states;

  • \$\Sigma\$ — input alphabet;

  • \$\Gamma\$ — stack alphabet;

  • \$q_0\$ — start state;

  • \$h_a\$ — accepting state;

  • \$\Delta\$ — symbol representing empty stack and end of input;

  • \$\delta\$ — transition function.

The transition function needs more detail.

The Transition Function

\$\delta : Q \times (\Gamma \cup \{\Delta\}) \times (\Sigma \cup \{\varepsilon,\Delta\}) \to \text{fin.\ subsets of } Q \times (\Gamma \cup \{\Delta\})^{\ast}\$

That is: look at the current state, the symbol on top of the stack, and (optionally) the input symbol; then update the state and replace the top symbol with a string of symbols:

  • pop = replace the top with nothing;

  • push = replace the top with itself followed by a new symbol.

Example

Example: \$\delta(q,A,a)=\{(p,AB)\}\$ means: in state \$q\$ with \$A\$ on top of the stack, upon reading a the PDA pushes \$B\$ and moves to state \$p\$.

Conversion from PDA to CFG

Idea: match each step of the derivation with a move of the PDA. This is not trivial!

We use the previous alternative representation. In addition, we assume:

(a) the stack of the PDA \$M\$ is empty iff \$M\$ is in the accepting state;

(b) each move is either a push of a single symbol or a pop of a single symbol.

The Variables are Triples

The variables of the CFG will be all ordered triples \$qAp\$ where \$q,p\$ are states of the PDA and \$A\$ is a symbol of the stack alphabet.
