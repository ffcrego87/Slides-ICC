Introduction

A grammar is a set of rules for putting strings together and so corresponds to a language.

Grammars

A grammar consists of:

  • a set of variables (nonterminals), one of which is the start variable (usual convenção: maiúsculas);

  • a set of terminals (alfabeto);

  • uma lista de productions (regras).

Example \$0^n 1^n\$

Here is a grammar:

\$S \rightarrow 0S1 \\ S \rightarrow \varepsilon\$

S é a única variável; os terminais são 0 e 1; há duas produções.

Using a Grammar

Uma produção permite, numa string que contém uma variável, substituir essa variável pelo lado direito (RHS) da produção.

Uma string de terminais \$w\$ é gerada pela gramática se, a partir da variável inicial, aplicarmos produções até obter \$w\$.

A sequência de strings obtidas é uma derivação de \$w\$.

Focamo-nos numa versão especial: context-free grammar (CFG). Uma linguagem é context-free se é gerada por uma CFG.

Example \$0^n 1^n\$ (continued)

\$S \rightarrow 0S1 \quad|\quad \varepsilon\$

A string 0011 é gerada com a derivação:

\$S \Rightarrow 0S1 \Rightarrow 00S11 \Rightarrow 0011\$

O símbolo | significa “ou”.

Example: Palindromes

Seja \$P\$ a linguagem dos palíndromos sobre o alfabeto \$\{a,b\}\$. Podemos obter uma CFG por decomposição recursiva:

  • retirar o primeiro e último símbolos de um palíndromo mantém um palíndromo;

  • envolver um palíndromo com o mesmo símbolo (frente e trás) mantém um palíndromo.

\$P \rightarrow aPa \;|\; bPb \;|\; \varepsilon\$

(Assim escrita, gera apenas palíndromos de comprimento par.)

Formal Definition

Uma CFG é um 4-tuplo \$(V, \Sigma, S, P)\$ onde:

  • \$V\$ é um conjunto finito de variáveis;

  • \$\Sigma\$ é um alfabeto finito de terminais;

  • \$S\$ é a variável inicial;

  • \$P\$ é um conjunto finito de produções, cada uma da forma

V \rightarrow (V \cup \Sigma)^{\ast}.

Further Examples: Even \$0\$’s

Uma CFG para todas as strings binárias com número par de 0’s.

  • Se o primeiro símbolo é 1, mantém-se “paridade par” de 0’s.

  • Se o primeiro é 0, avança até ao próximo 0; depois volta a par.

\$S \rightarrow 1S \;|\; 0A0S \;|\; \varepsilon \\ A \rightarrow 1A \;|\; \varepsilon\$

Alternate CFG for Even \$0\$’s

Outra CFG para a mesma linguagem. Se o primeiro é 0, o resto tem número ímpar de 0’s.

\$S \rightarrow 1S \;|\; 0T \;|\; \varepsilon \\ T \rightarrow 1T \;|\; 0S\$

Example

CFG para a linguagem regular do RE \$0 0^{\ast} 1 1^{\ast}\$ (tudo zeros seguido de tudo uns):

\$S \rightarrow CD \\ C \rightarrow 0C \;|\; 0 \\ D \rightarrow 1D \;|\; 1\$

Example: Complement

CFG para o complemento do RE \$0 0^{\ast} 1 1^{\ast}\$.

CFG não faz “e”, mas faz “ou”. Uma string não de forma \$0^i 1^j\$ com \$i,j>0\$ é uma das seguintes:

  • contém 10;

  • é só zeros;

  • é só uns.

Isto dá a CFG:

\$S \rightarrow A \;|\; B \;|\; C \\ A \rightarrow D\,10\,D \\ D \rightarrow 0D \;|\; 1D \;|\; \varepsilon \\ B \rightarrow 0B \;|\; 0 \\ C \rightarrow 1C \;|\; 1\$

Consistency and Completeness

Para verificar que a gramática corresponde à descrição, checar:

  1. Consistência: tudo o que a gramática gera encaixa na descrição.

  2. Completude: tudo o que a descrição permite é gerado pela gramática.

Example

Considere a CFG

\$S \rightarrow 0S1S \;|\; 1S0S \;|\; \varepsilon\$

A string 011100 é gerada por:

\$S \Rightarrow 0S1S \Rightarrow 01S \Rightarrow 011S0S \Rightarrow 0111S0S0S \Rightarrow 01110S0S \Rightarrow 011100S \Rightarrow 011100\$

O que contém esta linguagem? Claramente, todas as strings geradas têm igual número de 0’s e 1’s… Mas será que todas as strings com igual número de 0’s e 1’s são geradas?

Example — Argument for Completeness

Sim. Em qualquer string com igual número de 0’s e 1’s, a igualdade é atingida pela primeira vez num certo ponto.

Se a string começa por 0, a primeira vez que a contagem iguala acontece num 1. A parte entre esse 0 inicial e esse 1 é um exemplo de igualdade, e o sufixo também. Logo, podemos decompor como \$0 w 1 x\$ com \$w\$ e \$x\$ na linguagem.

A Silly Language CFG

CFG que gera frases como composição de sintagma nominal (NP) e verbal (VP):

\$S \rightarrow NP\;VP \\ NP \rightarrow \text{the}\;N \\ VP \rightarrow V\;NP \\ V \rightarrow \text{sings} \;|\; \text{eats} \\ N \rightarrow \text{cat} \;|\; \text{song} \;|\; \text{canary}\$

Gera “the canary sings the song”, mas também “the song eats the cat”. A gramática gera frases legais (sintaticamente), não necessariamente significativas.

Summary

Uma CFG consiste num conjunto de produções que substituem variáveis por cadeias de variáveis e terminais.

A linguagem de uma gramática é o conjunto de strings que ela gera.

Uma linguagem é context-free se existe uma CFG que a gera.

Derivation Trees and Regular Languages

Consideramos derivações especiais e representações (árvores de derivação) e ligamos CFGs a linguagens regulares.

Leftmost & Rightmost Derivations

Strings \$\alpha\$ yields string \$\beta\$, escrito \$\alpha \Rightarrow^{\ast} \beta\$, se é possível chegar de \$\alpha\$ a \$\beta\$ usando produções.

Uma derivação é a sequência de passos que leva a \$\beta\$.

  • leftmost derivation: em cada passo substitui-se a variável mais à esquerda.

  • rightmost derivation: análogo, mais à direita.

Derivation Trees

Numa árvore de derivação, a raiz é a variável inicial; nós internos são variáveis; folhas são terminais.

Os filhos de um nó interno (da esquerda para a direita) seguem o RHS da produção aplicada.

Example

Para a CFG de igualdade de 0’s e 1’s:

\$S \rightarrow 0S1S \;|\; 1S0S \;|\; \varepsilon\$

A derivação (esquerda-mais) de 011100 já vista é:

\$S \Rightarrow 0S1S \Rightarrow 01S \Rightarrow 011S0S \Rightarrow 0111S0S0S \Rightarrow 01110S0S \Rightarrow 011100S \Rightarrow 011100\$

Ambiguous Grammars

Uma gramática é unambiguous se existe uma única derivação leftmost (equivalentemente, uma única árvore de derivação) para cada string da linguagem.

A gramática de igualdade é ambígua:

\$S \rightarrow 0S1S \;|\; 1S0S \;|\; \varepsilon\$

(por exemplo, 0101 tem duas árvores possíveis.)

Grammars and Compilers

Compiladores usam gramáticas: validam o ficheiro ao encontrar a árvore de derivação.

Essa árvore alimenta a geração de código / análise semântica. Se a gramática é não-ambígua, a árvore é única.

Grammar for Arithmetic

Expressões aritméticas com apenas + e ×, variável inicial \$E\$:

\$E \rightarrow E + T \;|\; T \\ T \rightarrow T \times F \;|\; F \\ F \rightarrow (E) \;|\; \text{number}\$

Gera, e.g., \$1 + (3 + 2) \times 5\$ e \$1 + 2 \times 3\$.

Regular Languages Revisited

Theorem. Every regular language is generated by a context-free grammar.

Uma forma de provar: algoritmo para converter um RE em CFG (recursivo):

  • união: \$S \rightarrow A \;|\; B\$;

  • concatenação: \$S \rightarrow C D\$;

  • estrela de \$E\$: \$S \rightarrow E S \;|\; \varepsilon\$.

Example CFG for RE

Para o RE stem[(11 + 00)^{\ast} 11]:

Nível superior é concatenação de duas partes; a primeira é “ou” de duas peças:

\$S \rightarrow T U \\ U \rightarrow 11 \\ T \rightarrow T V \;|\; \varepsilon \\ V \rightarrow 00 \;|\; 11\$

Summary (reprise)

Cada string na linguagem tem uma derivação leftmost e uma árvore de derivação. Se estas são únicas para todas as strings, a gramática é unambígua.
