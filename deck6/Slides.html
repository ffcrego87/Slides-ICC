Introduction

A grammar is a set of rules for putting strings together and thus corresponds to a language.

Grammars

A grammar consists of:

  • a set of variables (nonterminals), one of which is the start variable (usual convention: uppercase);

  • a set of terminals (alphabet);

  • a list of productions (rules).

Example \$0^n 1^n\$

Here is a grammar:

\$S \rightarrow 0S1\$
\$S \rightarrow \varepsilon\$

S is the only variable; the terminals are 0 and 1; there are two productions.

Using a Grammar

A production allows, in a string that contains a variable, to replace that variable with the right-hand side (RHS) of the production.

A string of terminals \$w\$ is generated by the grammar if, starting from the start variable, we apply productions until we obtain \$w\$.

The sequence of strings obtained is a derivation of \$w\$.

We focus on a special version: a context-free grammar (CFG). A language is context-free if it is generated by a CFG.

Example \$0^n 1^n\$ (continued)

\$S \rightarrow 0S1 \quad|\quad \varepsilon\$

The string 0011 is generated with the derivation:

\$S \Rightarrow 0S1 \Rightarrow 00S11 \Rightarrow 0011\$

The symbol | means “or”.

Example: Palindromes

Let \$P\$ be the language of palindromes over the alphabet \$\{a,b\}\$. We can obtain a CFG by recursive decomposition:

  • removing the first and last symbols of a palindrome keeps a palindrome;

  • wrapping a palindrome with the same symbol (front and back) keeps a palindrome.

\$P \rightarrow aPa | bPb | \varepsilon\$

(As written, it generates only even-length palindromes.)

Formal Definition

A CFG is a 4-tuple \$(V, \Sigma, S, P)\$ where:

  • \$V\$ is a finite set of variables;

  • \$\Sigma\$ is a finite alphabet of terminals;

  • \$S\$ is the start variable;

  • \$P\$ is a finite set of productions, each of the form

\$V \rightarrow (V \cup \Sigma)^{\ast}.\$

Further Examples: Even \$0\$’s

A CFG for all binary strings with an even number of 0’s.

  • If the first symbol is 1, the “even parity” of 0’s remains.

  • If the first is 0, move forward until the next 0; then return to even.

\$S \rightarrow 1S | 0A0S | \varepsilon\$
\$A \rightarrow 1A | \varepsilon\$

Alternate CFG for Even \$0\$’s

Another CFG for the same language. If the first is 0, the rest has an odd number of 0’s.

\$S \rightarrow 1S | 0T | \varepsilon\$
\$T \rightarrow 1T | 0S\$

Example

CFG for the regular language of the RE \$0 0^{\ast} 1 1^{\ast}\$ (all zeros followed by all ones):

\$S \rightarrow CD\$
\$C \rightarrow 0C | 0\$
\$D \rightarrow 1D | 1\$

Example: Complement

CFG for the complement of the RE \$0 0^{\ast} 1 1^{\ast}\$.

A CFG cannot do “and”, but it can do “or”. A string not of the form \$0^i 1^j\$ with \$i,j>0\$ is one of the following:

  • contains 10;

  • is only zeros;

  • is only ones.

This gives the CFG:

\$S \rightarrow A | B | C\$
\$A \rightarrow D\,10\,D\$
\$D \rightarrow 0D | 1D | \varepsilon\$
\$B \rightarrow 0B | 0\$
\$C \rightarrow 1C | 1\$

Consistency and Completeness

To check that the grammar matches the description, verify:

  1. Consistency: everything the grammar generates fits the description.

  2. Completeness: everything allowed by the description is generated by the grammar.

Example

Consider the CFG

\$S \rightarrow 0S1S | 1S0S | \varepsilon\$

The string 011100 is generated by:

\$S \Rightarrow 0S1S \Rightarrow 01S \Rightarrow 011S0S \Rightarrow 0111S0S0S \Rightarrow 01110S0S \Rightarrow 011100S \Rightarrow 011100\$

What does this language contain? Clearly, all strings generated have an equal number of 0’s and 1’s… But are all strings with an equal number of 0’s and 1’s generated?

Example — Argument for Completeness

Yes. In any string with an equal number of 0’s and 1’s, equality is reached for the first time at a certain point.

If the string starts with 0, the first time the counts are equal occurs at a 1. The part between that initial 0 and that 1 is an example of equality, and so is the suffix. Thus, we can decompose it as \$0 w 1 x\$ with \$w\$ and \$x\$ in the language.

A Silly Language CFG

CFG that generates sentences as a composition of noun phrase (NP) and verb phrase (VP):

\$S \rightarrow NP VP\$
\$NP \rightarrow \text{the} N\$
\$VP \rightarrow V NP\$
\$V \rightarrow \text{sings} | \text{eats}\$
\$N \rightarrow \text{cat} | \text{song} | \text{canary}\$

Generates “the canary sings the song”, but also “the song eats the cat”. The grammar generates legal (syntactically correct) sentences, not necessarily meaningful ones.

Summary

A CFG consists of a set of productions that replace variables with strings of variables and terminals.

The language of a grammar is the set of strings it generates.

A language is context-free if there exists a CFG that generates it.

Derivation Trees and Regular Languages

We consider special derivations and representations (derivation trees) and link CFGs to regular languages.

Leftmost & Rightmost Derivations

String \$\alpha\$ yields string \$\beta\$, written \$\alpha \Rightarrow^{\ast} \beta\$, if it is possible to reach \$\beta\$ from \$\alpha\$ using productions.

A derivation is the sequence of steps that leads to \$\beta\$.

  • leftmost derivation: in each step, replace the leftmost variable.

  • rightmost derivation: analogous, but rightmost.

Derivation Trees

In a derivation tree, the root is the start variable; internal nodes are variables; leaves are terminals.

The children of an internal node (from left to right) follow the RHS of the production applied.

Example

For the CFG of equality of 0’s and 1’s:

\$S \rightarrow 0S1S | 1S0S | \varepsilon\$

The (leftmost) derivation of 011100 seen before is:

\$S \Rightarrow 0S1S \Rightarrow 01S \Rightarrow 011S0S \Rightarrow 0111S0S0S \Rightarrow 01110S0S \Rightarrow 011100S \Rightarrow 011100\$

Ambiguous Grammars

A grammar is unambiguous if there is a single leftmost derivation (equivalently, a single derivation tree) for each string in the language.

The equality grammar is ambiguous:

\$S \rightarrow 0S1S | 1S0S | \varepsilon\$

(for example, 0101 has two possible trees.)

Grammars and Compilers

Compilers use grammars: they validate the file by finding the derivation tree.

That tree feeds code generation / semantic analysis. If the grammar is unambiguous, the tree is unique.

Grammar for Arithmetic

Arithmetic expressions with only + and ×, start variable \$E\$:

\$E \rightarrow E + T | T\$
\$T \rightarrow T \times F | F\$
\$F \rightarrow (E) | \text{number}\$

Generates, e.g., \$1 + (3 + 2) \times 5\$ and \$1 + 2 \times 3\$.

Regular Languages Revisited

Theorem. Every regular language is generated by a context-free grammar.

One way to prove: algorithm to convert an RE into a CFG (recursive):

  • union: \$S \rightarrow A | B\$;

  • concatenation: \$S \rightarrow C D\$;

  • star of \$E\$: \$S \rightarrow E S | \varepsilon\$.

Example CFG for RE

For the RE \$(11 + 00)^{\ast} 11\$:

Top level is concatenation of two parts; the first is “or” of two pieces:

\$S \rightarrow T U\$
\$U \rightarrow 11\$
\$T \rightarrow T V | \varepsilon\$
\$V \rightarrow 00 | 11\$

Summary (reprise)

Each string in the language has a leftmost derivation and a derivation tree. If these are unique for all strings, the grammar is unambiguous.
